PLAN DE ITERACIONES - EXAMEN2
=============================

Objetivo general
----------------

Implementar un sistema cliente-servidor UDP en Python, siguiendo el contrato de
trabajo PED y el ciclo TDD pragmático:

RED -> GREEN -> REFACTOR

Cada entrega se realizará de forma separada para que pueda ser verificada y
subida individualmente por el alumno.

Tecnología y estructura prevista
--------------------------------

- Lenguaje: Python 3.
- Paradigma: Programación Orientada a Objetos.
- Testing: unittest.
- Comunicación: sockets UDP reales.
- Host por defecto: 127.0.0.1.
- Puerto por defecto: 16063.
- Código fuente: src/.
- Tests: tests/.
- Documentación final: README.txt e INSTALL.txt.
- Ejecución: mediante make.

Protocolo funcional
-------------------

Mensajes cliente -> servidor, codificados en UTF-8:

- BUSCAR <cadena>
- NUMERO
- SALIR

Respuestas servidor -> cliente:

- BUSCAR <cadena>:
  devuelve el número de líneas encontradas, un salto de línea y las líneas
  completas encontradas en /etc/passwd y /etc/services.

- NUMERO:
  devuelve una línea con OK y el número de búsquedas ejecutadas.

- SALIR:
  devuelve OK y finaliza el servidor.

- Cualquier otro mensaje:
  devuelve ERROR.
